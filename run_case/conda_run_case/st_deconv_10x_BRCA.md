# Reproducing Results: ST Deconvolution on 10x_BRCA — Conda Version

This document describes how to reproduce the **10x_BRCA spatial transcriptomics (ST) deconvolution** results using a **Conda environment**.

---

## 1) Download the `ST_10X_BRCA` package

Download the folder **`ST_10X_BRCA/`** from [https://doi.org/10.5281/zenodo.18495002](https://doi.org/10.5281/zenodo.18495002).

It contains **six sample subfolders**:

- `sample1/`
- `sample2/`
- `sample3/`
- `sample4/`
- `sample5/`
- `sample6/`

Each `sampleX/` folder contains three subfolders:

- `input/`  
  Input data for this case:
  - single-cell reference (`sc`, `.h5ad`)
  - spatial transcriptomics data (`st`, `.h5ad`)

- `st_model/`  
  **Pretrained model checkpoints** for the **10x_BRCA** case.

- `result_data/`  
  Reference results provided for comparison/validation.

### Input file naming convention (important)

- The **single-cell reference** is the same across all 6 datasets:  
  `sc_adata.h5ad`

- The **spatial** dataset is sample-specific and uses `_subX` to mark the sample:  
  `st_adata_sub_X.h5ad`  
  where `X ∈ {1,2,3,4,5,6}`.

Example (sample3):

- `ST_10X_BRCA/sample3/input/sc_adata.h5ad`
- `ST_10X_BRCA/sample3/input/st_adata_sub_3.h5ad`

## 2) Notes on reproducibility (methodological rationale)

Even with a fixed random seed, strict reproduction may be affected by:

1. **Platform-dependent numerical variation**  
   Parts of the pipeline rely on matrix factorization and low-level linear algebra routines. Numerical results can differ slightly across hosts due to differences in BLAS/LAPACK backends (e.g., MKL vs OpenBLAS), CPU vectorization, thread scheduling, and floating-point non-associativity. These small differences can propagate downstream and lead to measurable output deviations.

2. **Default Top-50 selection behavior**  
   The pipeline uses a default **Top-50 selection feature** step. When this set is recomputed, ties or near-ties may be resolved differently across environments or library versions, which can further contribute to differences in outputs.

### Recommendation for strict reproduction

To reproduce our reported results as closely as possible:

- **Reuse the provided pretrained checkpoints** (copy `st_model/` into your chosen output directory), and
- Run the pipeline with `--reproduce TRUE` (if your script supports it).

---

## 3) Choose your input and output folders

You only need to define two host directories:

- `CASE_DIR`: where you placed `ST_10X_BRCA/`
- `OUTPUT_DIR`: a writable folder where outputs will be saved

Example (replace with your own paths):

```bash
CASE_DIR="/path/to/ST_10X_BRCA"
OUTPUT_DIR="/path/to/your/output_folder"
```

Create the output directory:
```bash
mkdir -p "${OUTPUT_DIR}"
```


## 4) Run the reproduction script (Conda)
Example: 
```bash
SAMPLE="sample3"
SUB_ID="3"

SAMPLE_DIR="${CASE_DIR}/${SAMPLE}"
SAMPLE_OUT="${OUTPUT_DIR}/${SAMPLE}"

mkdir -p "${SAMPLE_OUT}"

# Copy pretrained checkpoints for this sample
cp -r "${SAMPLE_DIR}/st_model" "${SAMPLE_OUT}/st_model"

python st_deconv_10x_brca.py \
  --sc_adata "${SAMPLE_DIR}/input/sc_adata.h5ad" \
  --st_adata "${SAMPLE_DIR}/input/st_adata_sub_${SUB_ID}.h5ad" \
  --out_dir "${SAMPLE_OUT}" \
  --dataset_name "10x_sub${SUB_ID}"
  ```
Outputs will be written to:
```bash
${SAMPLE_DIR}
```

For other samples, just update:

```bash
SAMPLE="samplex"
SUB_ID="x"
```
accordingly.

## 5) ST reconstruction (st_mapping) from the deconvolution outputs (sample3)

After finishing ST deconvolution (Section 4), run **ST reconstruction** using the deconvolution outputs to map single cells back to spatial locations.

### 5.1 Overview

The ST reconstruction step takes:

- **sc_adata**: the single-cell reference (same as in deconvolution)
- **st_adata**: the ST AnnData **produced by the deconvolution step**, which contains the predicted cell-type fractions
- **annotation_key**: cell-type annotation column in `sc_adata.obs` (typically `cell_type`)
- **out_dir**: output directory (keep the same as before)
- **dataset_name**: same dataset name tag as in deconvolution (e.g., `10x_sub3`)

### 5.2 Run the reconstruction (sample3)

The reconstruction script **`st_10x_mapping.py`** accepts command-line arguments for flexibility. Use the same paths and dataset name as in the deconvolution step.

Set the dataset name (or reuse from Section 4):

```bash
DATASET_NAME="10x_sub${SUB_ID}"
```

Run the reconstruction:

```bash
python st_10x_mapping.py \
  --sc_adata "${SAMPLE_DIR}/input/sc_adata.h5ad" \
  --st_adata "${SAMPLE_OUT}/output/${DATASET_NAME}_st_adata.h5ad" \
  --out_dir "${SAMPLE_OUT}" \
  --dataset_name "${DATASET_NAME}"
```

**Key points:**

- `--sc_adata`: same single-cell reference as in deconvolution
- `--st_adata`: the ST AnnData file **generated by deconvolution** (in `${SAMPLE_OUT}/output/`)
- `--out_dir`: same output directory as before
- `--dataset_name`: same dataset tag (e.g., `10x_sub3`)

The script uses hardcoded parameters:
- `annotation_key = "cell_type"`
- `seed = 64`

### 5.3 Expected outputs

After successful completion, you will find the reconstructed spatial single-cell data in:

```
${SAMPLE_OUT}/${DATASET_NAME}_reconstructed_adata.h5ad
```

For sample3, this will be:
```
${SAMPLE_OUT}/10x_sub3_reconstructed_adata.h5ad
```

### 5.4 Run reconstruction for other samples

For other samples (e.g., sample2), simply update the corresponding variables:

```bash
SAMPLE="sample2"
SUB_ID="2"
DATASET_NAME="10x_sub${SUB_ID}"

SAMPLE_DIR="${CASE_DIR}/${SAMPLE}"
SAMPLE_OUT="${OUTPUT_DIR}/${SAMPLE}"
```
