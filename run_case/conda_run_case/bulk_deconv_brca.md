# Reproducing Results: Bulk Deconvolution on `BULK_BRCA` — Conda Version

This document describes how to reproduce the **bulk deconvolution** results for **BRCA bulk RNA** in `BULK_BRCA/` using a **Conda environment**.

The input package `BULK_BRCA/` contains **one task**:

- `BRCA` (one bulk cohort + one single-cell reference)

---

## 1) Download the `BULK_BRCA` package

Download the folder **`BULK_BRCA/`** from [https://doi.org/10.5281/zenodo.18495002](https://doi.org/10.5281/zenodo.18495002).

Folder structure (as provided):

- `BULK_BRCA/`
  - `input/`
    - `brca_sc_adata.h5ad` (single-cell reference)
    - `brca_bulk_adata.h5ad` (bulk cohort)
  - `model/`
    - pretrained checkpoints for reproducibility (if provided)
  - `result_data/`
    - reference results provided for comparison/validation (if provided)

### Input naming convention (important)

- Single-cell reference (sc, `.h5ad`):
  - `BULK_BRCA/input/brca_sc_adata.h5ad`
- Bulk cohort (bulk, `.h5ad`):
  - `BULK_BRCA/input/brca_bulk_adata.h5ad`

---

## 2) Notes on reproducibility

Even with a fixed random seed, strict reproduction may be affected by:

1. **Platform-dependent numerical variation**  
   Parts of the pipeline rely on matrix factorization and low-level linear algebra routines. Numerical results can differ slightly across hosts due to differences in BLAS/LAPACK backends (e.g., MKL vs OpenBLAS), CPU vectorization, thread scheduling, and floating-point non-associativity.

2. **Feature selection / tie-breaking differences**  
   Some internal steps may involve selecting top features. Ties or near-ties can be resolved differently across environments or library versions, contributing to small output differences.

## 3) Choose your case folder and output folder

Define:

- `CASE_DIR`: where you placed `BULK_BRCA/`
- `OUTPUT_DIR`: a writable folder where outputs will be saved

Example:

```bash
CASE_DIR="/path/to/BULK_BRCA"
OUTPUT_DIR="/path/to/your/output_folder"
```
Create the output directory:

```bash
mkdir -p "${OUTPUT_DIR}"
```

## 4) Example: reproduce BRCA bulk deconvolution
### 4.1 Set variables
```bash
DATASET_NAME="BRCA"

DATASET_DIR="${CASE_DIR}"
DATASET_OUT="${OUTPUT_DIR}/${DATASET_NAME}"

mkdir -p "${DATASET_OUT}"
```
4.2 Copy pretrained checkpoints into OUT_DIR (required)

Copy the pretrained checkpoints into the output directory that you will pass to --out_dir:
```bash
cp -r "${DATASET_DIR}/model" "${DATASET_OUT}/model"
```
### 4.3 Run the reproduction script

```bash
python bulk_deconv_brca.py \
  --sc_adata "${DATASET_DIR}/input/brca_sc_adata.h5ad" \
  --bulk_adata "${DATASET_DIR}/input/brca_bulk_adata.h5ad" \
  --out_dir "${DATASET_OUT}"
```

Outputs will be written under:

```bash
${OUTPUT_DIR}/BRCA/
```
## 5) Bulk mapping

This step runs **bulk mapping** using the output AnnData generated by the previous **bulk deconvolution** step.

### 5.1 Inputs

- Single-cell reference (same as deconvolution):
  - `BULK_BRCA/input/brca_sc_adata.h5ad`

- Deconvolution output bulk AnnData (generated by `bulk_deconv_brca.py` under `out_dir/output/`):
  - `${DATASET_OUT}/output/CID_4535_bulk_adata.h5ad`

> Naming consistency: the prefix `CID_4535` comes from the **hard-coded** `dataset_name="CID_4535"` inside `bulk_deconv_brca.py`.  
> If you changed that value in the script, the output filename will change accordingly.

Set the path:

```bash
DECONV_ADATA="${DATASET_OUT}/output/CID_4535_bulk_adata.h5ad"
```

### 5.2 Run the mapping script
```bash
python bulk_brca_mapping.py \
  --sc_adata "${DATASET_DIR}/input/brca_sc_adata.h5ad" \
  --deconv_adata "${DECONV_ADATA}" \
  --out_dir "${DATASET_OUT}"
```

Outputs will be written under:

```bash
${OUTPUT_DIR}/BRCA/
```
