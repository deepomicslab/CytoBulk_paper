# Reproducing Results: **Bulk Deconvolution** on `human_bulk` (BULK_Human_sc) — Conda Version

This document describes how to reproduce the **bulk deconvolution** results for the **human_bulk** case (package: `BULK_Human_sc/`) using a **Conda environment**.

The input package `BULK_Human_sc/` contains **one task** (one bulk cohort + one single-cell reference):


---

## 1) Download the `BULK_Human_sc` package

Download the folder **`BULK_Human_sc/`** from [https://doi.org/10.5281/zenodo.18495002](https://doi.org/10.5281/zenodo.18495002).

Folder structure (as provided):

- `BULK_Human_sc/`
  - `input/`
    - `A36_sample.h5ad` (single-cell reference)
    - `simulated_bulk.h5ad` (bulk / simulated bulk)
  - `model/`  
    **Pretrained model checkpoints** for reproduction.
  - `result_data/`  
    Reference results for comparison/validation.

> Note: this tutorial assumes `input/`, `model/`, and `result_data/` are directly under `BULK_Human_sc/`.

---

## 2) Notes on reproducibility

Even with a fixed random seed, strict reproduction may be affected by:

1. **Platform-dependent numerical variation**  
   Parts of the pipeline rely on low-level linear algebra and deep-learning kernels. Numerical results can differ slightly across hosts due to differences in BLAS/LAPACK backends (MKL vs OpenBLAS), CPU/GPU kernels, thread scheduling, and floating-point non-associativity.

2. **Library version differences**  
   Minor version changes in `cytobulk`, `scanpy/anndata`, or `torch` can lead to small deviations (e.g., tie-breaking during feature selection, internal defaults).

### Recommendation for strict reproduction

To reproduce our reported results as closely as possible:

- **Reuse the provided pretrained checkpoints** by copying `model/` into your chosen output directory (the folder you pass to `--out_dir`).

Your Conda script `bulk_deconv_human_bulk.py` uses a fixed `dataset_name="human_bulk"` and will write outputs using that name; ensuring `out_dir/model/` exists makes the run consistent with the intended “reproduce” behavior.

---

## 3) Choose your input and output folders

You only need to define:

- `PKG_DIR`: where you placed `BULK_Human_sc/`
- `DATA_DIR`: where the input `.h5ad` files are stored (`${PKG_DIR}/input`)
- `OUTPUT_DIR`: a writable folder where outputs will be saved

Example (replace with your own paths):

```bash
PKG_DIR="/path/to/BULK_Human_sc"
DATA_DIR="${PKG_DIR}/input"
OUTPUT_DIR="/path/to/your/output_folder"
```
Create the output directory:

```bash
mkdir -p "${OUTPUT_DIR}"
```

## 4) Example: reproduce BRCA bulk deconvolution
### 4.1 Set variables
```bash
DATASET_NAME="Human_sc"
DATASET_DIR="${PKG_DIR}"
DATASET_OUT="${OUTPUT_DIR}/${DATASET_NAME}"
mkdir -p "${DATASET_OUT}"
```
4.2 Copy pretrained checkpoints into OUT_DIR (required)

Copy the pretrained checkpoints into the output directory that you will pass to --out_dir:
```bash
cp -r "${DATASET_DIR}/model" "${DATASET_OUT}/model"
```
### 4.3 Run the reproduction script

```bash
python bulk_deconv_human_bulk.py \
  --sc_adata "${DATA_DIR}/A36_sample.h5ad" \
  --bulk_adata "${DATA_DIR}/simulated_bulk.h5ad" \
  --out_dir "${DATASET_OUT}" \
```

Outputs will be written under:

```bash
${OUTPUT_DIR}/Human_sc/
```
## 5) Bulk mapping

This step runs **bulk mapping** using the output AnnData generated by the previous **bulk deconvolution** step.

### 5.1 Inputs

- Single-cell reference (same as deconvolution):
  - ${DATA_DIR}/A36_sample.h5ad

- Deconvolution output bulk AnnData (generated by `bulk_deconv_human_bulk.py` under `out_dir/output/`):
  - ${DATASET_OUT}/output/human_bulk_bulk_adata.h5ad

> Naming consistency: the prefix `human_bulk` comes from the **hard-coded** `dataset_name="human_bulk"`.
> If you changed that value in the script, the output filename will change accordingly.

### 5.2 Run the mapping script
