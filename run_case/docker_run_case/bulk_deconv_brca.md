# Reproduce **bulk deconvolution** results for **BULK_BRCA** using Docker

This document describes how to reproduce the **bulk deconvolution** (and optional **bulk mapping**) results for **BRCA bulk RNA** using our **Docker image**.

The input package `BULK_BRCA/` contains **one task** (one bulk cohort + one single-cell reference):

- `BRCA`

---

## 1) Download the `BULK_BRCA` package

Download the folder **`BULK_BRCA/`** from **`XXXX`**.

Folder structure (as provided):

- `BULK_BRCA/`
  - `input/`
    - `brca_sc_adata.h5ad` (single-cell reference)
    - `brca_bulk_adata.h5ad` (bulk cohort)
  - `model/` (pretrained checkpoints for reproduction, if provided)
  - `result_data/` (reference results for validation, if provided)

> Note: this tutorial assumes `input/`, `model/`, and `result_data/` are directly under `BULK_BRCA/`.

---

## 2) Choose your input and output folders (host-side)

Define:

- `CASE_DIR`: where you placed `BULK_BRCA/`
- `OUTPUT_DIR`: a writable folder to store outputs

```bash
CASE_DIR="/path/to/BULK_BRCA"
OUTPUT_DIR="/path/to/your/output_folder"
mkdir -p "${OUTPUT_DIR}"
```

Set dataset variables:
## 3) Set variables
```bash
DATASET_NAME="BRCA"

DATASET_DIR="${CASE_DIR}"
DATASET_OUT="${OUTPUT_DIR}/${DATASET_NAME}"
mkdir -p "${DATASET_OUT}"
```

## 4)  Reproduce: BRCA bulk deconvolution

### 4.1 Copy pretrained checkpoints (required)
```bash
cp -r "${DATASET_DIR}/model" "${DATASET_OUT}/model"
```

### 4.2 Run Docker
```bash
docker run --rm -it \
  -e PYTHONUNBUFFERED=1 \
  -e HOST_UID="$(id -u)" \
  -e HOST_GID="$(id -g)" \
  -v "${DATASET_DIR}/input":/inputs:ro \
  -v "${DATASET_OUT}":/outputs \
  kristawang/cytobulk:1.0.0 \
  bulk_deconv \
  --sc "/inputs/brca_sc_adata.h5ad" \
  --bulk "/inputs/brca_bulk_adata.h5ad" \
  --annotation_key "celltype_minor" \
  --out_dir "/outputs/" \
  --dataset_name "CID_4535" \
  --n_cell 500 \
  --seed 64 \
  --downsampling True
  --reproduce True
```

Outputs will be written under:
```bash
${OUTPUT_DIR}/BRCA/
```

## 4) Parameter explanations 

Below are the parameters used in the TCGA reproduction commands and what they mean.


- `--sc <path>`  
  Single-cell reference `.h5ad` file (inside container).

- `--bulk <path>`  
  Bulk dataset `.h5ad` file (inside container).

- `--annotation_key <str>`  
  Column name in `sc_adata.obs` that stores the cell-type labels. 

- `--dataset_name <str>`  
  Output prefix / project identifier used for organizing outputs.

- `--out_dir <path>`  
  Output folder (inside container). In this tutorial always use `"/outputs/"`.

- `downsampling <True|False>`  
  Set True to downsample single cell. 


- `--seed <int>`  
  Random seed used for data simulation and training control. We use `64` as in the scripts.

- `--reproduce`  
  Load pretrained checkpoints from `out_dir/model/` and run in reproduction mode.  
  This is why you must copy `model/` into the output directory before running.


- `--n_cell <int>`  
  Number of cells used when simulating pseudo-bulk mixtures during training.  
  Here: `500`.

---

## 5) Bulk mapping

This step runs **bulk mapping** using the output AnnData generated by the previous **bulk_deconv** step.

### 5.1 Inputs

- Single-cell reference (same as deconvolution):
  - `/inputs/brca_sc_adata.h5ad`

- Deconvolution output bulk AnnData (generated in the previous step under `out_dir/output/`):
  - `/outputs/output/<DECONV_DATASET_NAME>_bulk_adata.h5ad`

> Naming consistency: `<DECONV_DATASET_NAME>` is exactly the `--dataset_name` you used in **bulk_deconv**.  
> In this tutorial we used `--dataset_name "human_bulk"`, so the file is:
> `/outputs/output/human_bulk_bulk_adata.h5ad`.

### 5.2 Run Docker
```bash
docker run --rm -it \
  -e PYTHONUNBUFFERED=1 \
  -e HOST_UID="$(id -u)" \
  -e HOST_GID="$(id -g)" \
  -v "${DATASET_DIR}/input":/inputs:ro \
  -v "${DATASET_OUT}":/outputs \
  kristawang/cytobulk:1.0.0 \
  bulk_mapping \
  --sc "/inputs/brca_sc_adata.h5ad" \
  --bulk "/outputs/output/CID_4535_bulk_adata.h5ad" \
  --annotation_key "celltype_minor" \
  --out_dir "/outputs/" \
  --dataset_name "brca_bulk" \
  --n_cell 1000 \
  --seed 64 \
  --sub_sample \
  --sample_key "orig.ident" \
  --keep_values CID4471 CID44971 CID4290A CID4530N CID4515 CID4535 CID44041
```

Naming consistency: the --bulk file name prefix comes from the deconv --dataset_name (CID_4535 here).
