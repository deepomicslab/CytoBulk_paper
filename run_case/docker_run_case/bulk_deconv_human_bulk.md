This document describes how to reproduce the **bulk deconvolution** results for **Human_sc** bulk RNA using our **Docker image**.

The input package `BULK_Human_sc/` contains **one task** (one bulk cohort + one single-cell reference):

- `Human_sc`

---

## 1) Download the `BULK_Human_sc` package

Download the folder **`BULK_Human_sc/`** from **`XXXX`**.

Folder structure (as provided):

- `BULK_Human_sc/`
  - `input/`
    - `A36_sample.h5ad` (single-cell reference)
    - `simulated_bulk.h5ad` (bulk / simulated bulk)
  - `model/` (pretrained checkpoints for reproduction)
  - `result_data/` (reference results for validation)

> Note: this tutorial assumes `input/`, `model/`, and `result_data/` are directly under `BULK_Human_sc/`.

---

## 2) Choose your input and output folders (host-side)

Define:

- `CASE_DIR`: where you placed `BULK_Human_sc/`
- `OUTPUT_DIR`: a writable folder to store outputs

```bash
CASE_DIR="/path/to/BULK_Human_sc"
OUTPUT_DIR="/path/to/your/output_folder"
mkdir -p "${OUTPUT_DIR}"
```

Set dataset variables:
### 3.1 Set variables
```bash
DATASET_NAME="Human_sc"

DATASET_DIR="${CASE_DIR}"
DATASET_OUT="${OUTPUT_DIR}/${DATASET_NAME}"
mkdir -p "${DATASET_OUT}"
```

## 3) Reproduce: Human Bulk

### 3.1 Copy pretrained checkpoints (required)
```bash
cp -r "${DATASET_DIR}/model" "${DATASET_OUT}/model"
```

### 3.2 Run Docker
```bash
docker run --rm -it \
  -e PYTHONUNBUFFERED=1 \
  -e HOST_UID="$(id -u)" \
  -e HOST_GID="$(id -g)" \
  -v "${DATASET_DIR}/input":/inputs:ro \
  -v "${DATASET_OUT}":/outputs \
  kristawang/cytobulk:1.0.0 \
  bulk_deconv \
  --sc "/inputs/A36_sample.h5ad" \
  --bulk "/inputs/simulated_bulk.h5ad" \
  --annotation_key "Manually_curated_celltype" \
  --out_dir "/outputs/" \
  --dataset_name "human_bulk" \
  --n_cell 500 \
  --specificity False \
  --seed 64 \
  --reproduce True
```

Outputs will be written under:
```bash
${OUTPUT_DIR}/Human_sc/
```

## 4) Parameter explanations 

Below are the parameters used in the TCGA reproduction commands and what they mean.


- `--sc <path>`  
  Single-cell reference `.h5ad` file (inside container).

- `--bulk <path>`  
  Bulk dataset `.h5ad` file (inside container).

- `--annotation_key <str>`  
  Column name in `sc_adata.obs` that stores the cell-type labels.  


- `--out_dir <path>`  
  Output folder (inside container). In this tutorial always use `"/outputs/"`.


  
- `--dataset_name <str>`  
  Output prefix / project identifier used for organizing outputs.

### Reproduction / randomness

- `--seed <int>`  
  Random seed used for data simulation and training control. We use `64` as in the scripts.

- `--reproduce`  
  Load pretrained checkpoints from `out_dir/model/` and run in reproduction mode.  
  This is why you must copy `model/` into the output directory before running.


- `--n_cell <int>`  
  Number of cells used when simulating pseudo-bulk mixtures during training.  
  Here: `100`.

- `--specificity <True|False>`  
  Whether to use the **specificity strategy** when **simulating training mixtures** (pseudo-bulk generation).  
  - `True`: simulate training data using the specificity-based strategy.  
  - `False`: simulate training data using **random** mixing only.  

For **randomly simulated bulk** data (like this dataset), we recommend setting `--specificity False`.

---

## 5) Bulk mapping

This step runs **bulk mapping** using the output AnnData generated by the previous **bulk_deconv** step.

### 5.1 Inputs

- Single-cell reference (same as deconvolution):
  - `/inputs/A36_sample.h5ad`

- Deconvolution output bulk AnnData (generated in the previous step under `out_dir/output/`):
  - `/outputs/output/<DECONV_DATASET_NAME>_bulk_adata.h5ad`

> Naming consistency: `<DECONV_DATASET_NAME>` is exactly the `--dataset_name` you used in **bulk_deconv**.  
> In this tutorial we used `--dataset_name "human_bulk"`, so the file is:
> `/outputs/output/human_bulk_bulk_adata.h5ad`.

### 5.2 Run Docker
```bash
docker run --rm -it \
  -e PYTHONUNBUFFERED=1 \
  -e HOST_UID="$(id -u)" \
  -e HOST_GID="$(id -g)" \
  -v "${DATASET_DIR}/input":/inputs:ro \
  -v "${DATASET_OUT}":/outputs \
  kristawang/cytobulk:1.0.0 \
  bulk_mapping \
  --sc "/inputs/A36_sample.h5ad" \
  --bulk "/outputs/output/human_bulk_bulk_adata.h5ad" \
  --annotation_key "Manually_curated_celltype" \
  --out_dir "/outputs/" \
  --dataset_name "human_bulk" \
  --n_cell 500 \
  --seed 64
  --multiprocessing False
```
